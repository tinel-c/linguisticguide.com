## ğŸ” File Permissions Guide

### Recommended Permissions

| Item | Permission | Numeric | Explanation |
|------|-----------|---------|-------------|
| Directories | `rwxr-xr-x` | 755 | Owner: read/write/execute, Others: read/execute |
| HTML files | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |
| CSS files | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |
| JS files | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |
| PHP files | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |
| Images | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |
| .htaccess | `rw-r--r--` | 644 | Owner: read/write, Others: read-only |

### Setting Permissions in FileZilla

1. Right-click on file/folder
2. Select "File Permissions"
3. Enter numeric value (e.g., 644)
4. Click OK

### Setting Permissions in cPanel

1. Open File Manager
2. Right-click file/folder
3. Select "Change Permissions"
4. Check appropriate boxes or enter numeric value
5. Click "Change Permissions"

## ğŸ› Troubleshooting

### Issue: Contact Form Not Working

**Solution 1**: Check PHP handler email
```php
// In contact-handler.php
define('RECIPIENT_EMAIL', 'your-actual-email@example.com');
```

**Solution 2**: Test PHP file directly
Visit: `https://linguisticguide.com/contact-handler.php`
Should show: Method not allowed (that's correct)

**Solution 3**: Check file permissions
- contact-handler.php should be 644

### Issue: .htaccess Not Working

**Solution**: Check if file uploaded correctly
- File must be named exactly `.htaccess` (with dot)
- File must be in public_html root
- Permission should be 644

### Issue: Images Not Loading

**Solution**: Check paths
- Images should be in assets/ folder
- Paths in HTML should be relative: `./assets/image.jpg`
- Check file names match (case-sensitive)

### Issue: HTTPS Not Working

**Solution**: Run AutoSSL
1. cPanel â†’ SSL/TLS Status
2. Run AutoSSL
3. Wait 15 minutes
4. Clear browser cache
5. Try again

### Issue: 500 Internal Server Error

**Causes**:
- Incorrect .htaccess syntax
- Wrong file permissions
- PHP errors

**Solution**:
1. Rename .htaccess temporarily
2. Check if site works
3. If yes, problem is in .htaccess
4. Check error logs in cPanel
